# 신용카드 이상거래(사기) 탐지

## 1. 문제 정의
- 무엇을 푸는가: 신용카드 거래 데이터에서 사기 거래를 탐지하는 이진 분류 문제다. 전체의 약 0.29%만이 사기인 극심한 클래스 불균형 상황에서, 소수의 사기를 정확히 가려내는 것이 목표다.
- 왜 어려운가: 불균형이 평가와 학습을 양쪽에서 망가뜨린다. 평가 면에서는 정상이 99.71%를 차지해 "모두 정상"이라 찍어도 accuracy가 99.71%가 나오므로 정확도가 거짓말을 한다. 학습 면에서는 모델이 사기 예제를 너무 적게 봐서, 사기 패턴을 배우기보다 "전부 정상"이라 답하는 게으른 해법으로 도망가버린다.

## 2. 데이터
- 출처: Kaggle Credit Card Fraud Detection
- 규모: 약 5.4만 건, 사기 153건(약 0.29%)
- 특징: V1~V28(PCA 익명화), Time, Amount / 타깃 Class

## 3. 평가지표를 왜 그렇게 골랐나
-왜 accuracy는 쓸모없는가:

이 데이터셋은 99.71%가 정상 거래인 극단적 불균형 구조다. accuracy = (TP+TN)/전체인데, 정상 더미(TN)가 5.4만 건에 달해 분자를 압도한다. 그 결과 "모든 거래를 정상이라 찍는" 게으른 모델조차 accuracy가 99.71%에 이른다 — 사기를 전부 놓쳤음에도(FN 폭증) 점수가 멀쩡한 것이다. 즉 accuracy는 우리가 실제로 관심 있는 소수 클래스(사기) 탐지 능력을 거대한 다수 클래스에 묻어버린다. 반면 precision과 recall은 계산에 TN을 전혀 쓰지 않아 사기에 대해 정직하게 말한다. 따라서 나는 accuracy를 폐기하고 precision·recall·PR-AUC를 핵심 지표로 채택했다.
	
-나는 recall / precision 중 recall을 더 중시했다:

사기 탐지에서 FN(사기를 놓침)은 빼앗긴 돈·보상·신뢰 하락으로 이어져, FP(헛경보로 정상 고객을 막음)의 비용보다 크다고 판단했기 때문이다. 헛경보를 어느 정도 감수하더라도 진짜 사기는 놓치지 않는 쪽을 택했다.
	
## 4. 시도한 방법과 비교표  ← 4·5주차 표
| 방법 | precision | recall | F1 | PR-AUC |
|---|---|---|---|---|
| 베이스라인(로지스틱) | 0.8289 | 0.6429 | 0.7241 | 0.7432 |
| 랜덤포레스트 | 0.9412 | 0.8163 | 0.8743 | 0.8734 |
| class_weight | 0.0610 | 0.9184 | 0.1144 | 0.7159 |
| SMOTE | 0.0580 | 0.9184 | 0.1092 | 0.7249 |
| 신경망(MLP) | 0.1238 | 0.8878 | 0.2172 | 0.7708 |

## 5. 결과 해석
-내 목표 기준 가장 나은 방법 + 이유

내 목표(recall 우선)만 보면 class_weight와 SMOTE가 recall 0.92로 가장 높다. 그러나 그 대가로 precision이 0.06까지 무너져, 사기 경보의 94%가 헛경보인 상태가 된다 — 실무에서 이 정도 오경보율은 콜센터 마비와 고객 이탈을 부르므로 받아들이기 어렵다. 반면 랜덤포레스트는 recall 0.82로 약간 낮지만 precision 0.94를 함께 유지해, recall과 precision의 균형 면에서 가장 우수하다. 따라서 나는 "recall을 중시하되 precision을 실무 가능 수준으로 지키는" 기준에서 랜덤포레스트를 최선의 모델로 택했다. recall을 절대적으로 끌어올려야 하는 상황이라면 class_weight를 쓰되, 임계값 조정으로 precision을 함께 관리하는 후속 작업이 필요하다.

-깊은 관찰: recall과 PR-AUC가 어떻게 달리 움직였나(operating point vs 랭킹 능력)

주목할 점은 class_weight·SMOTE에서 recall은 0.64→0.92로 급등했지만 PR-AUC는 0.74→0.72로 거의 변하지 않았다는 것이다. P/R/F1은 기본 임계값(0.5) 한 지점의 성적이고, class_weight·SMOTE는 결정 경계를 옮겨 마치 임계값을 낮춘 것과 같은 효과를 낸다. 반면 PR-AUC는 모든 임계값을 훑은 랭킹 능력을 요약한다. 즉 recall 급등은 "모델이 사기를 근본적으로 더 잘 구분하게 된 것"이 아니라 "경보 기준을 헐겁게 한 것"에 가깝다. PR-AUC가 그대로라는 것은 사기를 정상 위로 랭킹하는 본질적 능력은 거의 변하지 않았음을 뜻한다.
	
-신경망 vs 랜덤포레스트:

신경망(MLP)은 PR-AUC 0.77로 베이스라인보다는 나았으나, 랜덤포레스트(0.87)에는 미치지 못했다. 이는 실패가 아니라 예견된 결과다. 우리 데이터는 열 순서나 공간 구조가 의미를 갖지 않는 정형 데이터(표)이고, 이런 데이터에서는 특성을 하나씩 분기로 다루는 트리 기반 모델이 모든 특성을 뒤섞어 학습하는 신경망보다 유리한 경우가 많다. 또 신경망은 많은 파라미터를 채우기 위해 대량의 데이터를 요구하는데, 5만여 건은 그 강점을 살리기에 충분치 않다. "신경망이 항상 이기는 것은 아니다" — 문제와 데이터 성격에 맞는 도구를 고르는 것이 중요하다는 교훈을 직접 확인했다.

## 6. 배운 점
-데이터 누수(스케일러·SMOTE를 train에만)

평가의 신뢰성은 test를 얼마나 깨끗하게 지키느냐에 달려 있음을 배웠다. 스케일러는 train에만 fit하고 test에는 그 기준을 적용만 해야 하며, SMOTE 같은 오버샘플링도 train에만 적용해야 한다. 만약 분리 전 전체 데이터에 이런 처리를 하면, test의 정보가 train에 새어 들어가(누수) 점수가 비현실적으로 부풀려진다. test는 "처음 보는 미래 거래"를 흉내 내야 하므로 끝까지 100% 원본으로 남겨야 한다는 원칙을 체화했다.
	
-학습 루프 네 줄의 의미

sklearn의 model.fit() 한 줄 안에 숨어 있던 과정을 PyTorch에서 직접 펼쳐보며 학습의 본질을 이해했다. forward(예측) → loss(틀린 정도 측정) → backward(각 가중치의 책임을 거꾸로 계산) → step(가중치를 조금씩 수정)이 한 사이클이고, 이것을 수많은 배치와 에포크에 걸쳐 반복하면 모델 내부의 가중치가 점점 더 나은 값으로 다듬어진다. "모델이 학습한다"는 것이 결국 내부 숫자(가중치)가 바뀌는 일이라는 걸 코드로 확인했다.
	
-지표를 표면이 아니라 비용 관점에서 읽는 법

높은 점수가 곧 좋은 모델이 아님을 배웠다. accuracy 99.7%는 사기를 전부 놓쳐도 나오는 허수였고, recall 0.92는 precision 0.06이라는 헛경보 폭탄을 가린 착시였다. 각 지표를 FN(사기 놓침)과 FP(헛경보)의 실제 비용으로 번역해 읽어야 비로소 의미가 생긴다. 특히 recall이 급등해도 PR-AUC가 그대로일 수 있다는 관찰을 통해, "지표가 좋아진 것"과 "결정 경계만 옮긴 것"을 구분하는 눈을 길렀다.
	
-다음에 더 해보고 싶은 것

 -임계값 최적화: class_weight는 recall이 높지만 precision이 무너졌다. 0.5 기본 임계값 대신 비용을 반영한 임계값을 탐색하면, recall을 유지하면서 precision을 실무 가능 수준으로 끌어올릴 수 있을지 실험해보고 싶다.
	
 -XGBoost/LightGBM 시도: 정형 데이터에서 랜덤포레스트가 강했으니, 같은 트리 계열의 부스팅 모델이 더 나은 성능을 낼지 확인해보고 싶다.
	
 -비용 기반 평가 지표 도입: FN과 FP에 실제 금액 비용을 매겨, 정확도나 F1이 아니라 "총 비용 최소화" 관점에서 모델을 다시 비교해보고 싶다 (산업공학적 접근).
	
 -SMOTE 비율 조정: 완전한 1:1 균형(sampling_strategy=1.0) 대신 더 보수적인 비율로 늘렸을 때 precision-recall 균형이 어떻게 달라지는지 실험해보고 싶다.
